Nuclear Cybersecurity

Introduction

Cybersecurity protects computers and networks from people or software that want to break in, steal data, or make systems fail. In nuclear systems, digital faults can quickly become safety events — loss of cooling, incorrect alarms, or corrupted control logic.

  • Confidentiality — protect secrets
  • Integrity — ensure commands and data are correct
  • Availability — systems must work when needed

Operation

Core components:

  • Sensors & actuators: measure/change physical variables
  • PLCs / RTUs / DCS: control systems in real-time
  • HMI / SCADA / Historian: operator interfaces & logs
  • Engineering workstations: design/upload logic

OT protocols often lack authentication/encryption; bridging legacy and modern protocols creates vulnerabilities. Nuclear networks use layered zoning (Purdue/ISA-IEC), DMZs, and data diodes to reduce lateral movement.

Breaches

  1. Reconnaissance: gather info on networks & personnel
  2. Initial access: phishing, vulnerable services, removable media
  3. Persistence & lateral movement: backdoors, credentials
  4. Manipulate OT: alter PLC logic, sensor readings, alarms
  5. Hide tracks: tamper with logs, stealth techniques

Defence

  • Segment networks, remove unnecessary IT–OT connections
  • Block autorun and scan removable media
  • Vendor access: MFA, logging, time-limited sessions
  • Offline backups of PLC/DCS logic
  • Whitelisting executables and firmware signing
  • OT-aware IDS/IPS and anomaly detection

Glossary

  • Asset Inventory: list of hardware, software, firmware
  • Availability: system works when required
  • Data Diode: one-way hardware data flow
  • Defense-in-Depth: layered technical, procedural, physical controls
  • Zero-Trust: no implicit trust; authenticate everything
